git Teknolojisi

  1. 1. git nedir?

        git, yazdığımız projeleri ve uygulamaları, bilgisayarımızda ya da harici disklerde değil de internet üzerinde tutmamızı ve yönetmemizi sağlayan bir versiyon kontrol sistemidir. GNU Genel Kamu Lisansı’nın 2. sürümüyle lisanslanmış bir özgür yazılımdır. Linus Torvalds tarafından tasarlanıp geliştirilmiştir.

        git, uzun çalışmalar sonucu geliştirdiğimiz ürünlerin hata yaptığımızda ve yaptığımız hatayı geri döndürmek gerektiğinde, projede çalışan kişi sayısı 1'den fazla olduğunda, projede yaptığımız değişikliklerin kontrollü olması gerektiğinde işe yarayan bir araçtır.

    git ile çalışırken, kaydettiğimiz dosyadan ayrı olarak bir de kaydedilen verileri git için ayrı olarak onaylanarak ayrı şekilde ve kontrollü olarak kaydederiz. Buna git commit denir.

    git versiyon kontrolünü ve takibini sizin adınıza otomatik olarak yapar. Siz şimdiki versiyondan memnun olmadığınızda, hatalarınızdan kurtulamadığınızda istediğiniz bir eski versiyona geri dönebilirsiniz. git'in en yaygın kullanım alanı ise grup çalışmalarıdır. Üzerinde birden fazla kişinin çalıştığı projelerin, takip ve yönetim açısından zorluğundan doğan ihtiyaca hizmet eden git projenin büyüklüğü farketmeksizin çoklu ya da uzaktan çalışmayı, projeyi yönetmeyi kolaylaştırır.

  2. 2. git nasıl kullanılır?

    Öncelikle git çevrimiçi kullanılması zorunlu olan bir şey değildir. Yerel olarak, kendi kişisel bilgisayarınızda da git kullanabilirsiniz. Ancak bu yedekleme mantığına aykırı ve tehlikelidir. Üzerinde uzun bir süre çalıştığınız projeyi elektrik kesintisi ya da elektronik bir arıza sebebiyle kaybetmek istemezsiniz :)

    git kullanmak için ilk olarak git'i yüklemeliyiz.

    Mac için git

    git'i Mac'e kurmanın birkaç yolu vardır. Öncelikle Mac'inizde git'in kurulu olup olmadığını öğrenmek için git --version komutunu terminal'e yazmanız gerek. Eğer output git version 2.7.0 (Apple Git-66) gibi bir şey olarak dönerse Mac'inizde git kurulu demektir.

    Eğer kurulu değilse, buradan en son versiyonunu indirip kurabiliriz.

    Windows için git

    Windows için git kurulumunu, buradan en son versiyonunu indirip yapabiliriz.

    Projemizi başlattıktan sonra ya da halihazırda olan projemizi versiyon kontrolle güçlendirmek istediğimizde, console(Windows) ya da terminal(Mac / Linux)'i açıp projemizin olduğu dizine gidiyoruz. Örneğin bu C:\Users\USER\Documents\Projelerim\git-practice olsun.

    terminale:
    $ cd C:\Users\USER\Documents\Projelerim\git-practice

    yazıp Enter'a basarak bu dizine gidebiliriz. Projemizin dizinine gittikten sonra yine terminale git init yazıp Enter'a bastığımızda projemizin git yapısı otomatik olarak oluşturulacaktır. İlk oluşturulduğunda git dizinimizin içi boştur. git dizinimizi

    $ git add .

    komutuyla proje dosyalarımızı stage dediğimiz aşamaya getirerek doldurmaya başlayabiliriz. stage aşaması bizim versiyon kontrolü yapılmasını istediğimiz dosyaları git'in görüş alanına almamız gibi düşünülebilir.

    Artık eklediğimiz dosyaları commit ederek projemizi git için kalıcı hale getirebiliriz. Bunun için:

    $ git commit -m "İlk commit mesajımız, proje dosyalarımızı git'e ekledik"

    komutunu kullanırız. buradaki -m "message" kelimesinin kısaltılmışıdır. Komutlarda kısa çizgi ile verilen bu tür opsiyonlara flag denir. -m den sonra yazdığımız "İlk commit mesajımız, proje dosyalarımızı git'e ekledik" ise bizim bundan aylar yıllar sonra bile baktığımızda o tarih ve saatte ne iş yaptığımızı anlayabilmemiz için yazdığımız ufak mesajlardır. her commit'te bu mesajın açık ve anlaşılır bir biçimde yazılması güzel bir uygulamadır.

  3. 3. GitHub nedir?

    GitHub versiyon kontrol sistemi olarak git kullanan projeler için depolama hizmeti sunan bir web servisidir. Proje dosyalarımızı uzak bir dizinde depolayarak daha güvenli bir şekilde yedek alabilir, arıza ve benzeri durumlarda güvende olabiliriz. Bunun için öncelikle GitHub hesabı oluşturmamız gerekiyor. GitHub.com'a giderek bir hesap oluşturuyoruz. Ardından anasayfada "New Repository" butonuna tıklayarak ya da sağ üstteki + ya tıklayarak "New Repository" seçeneğini seçerek ve yeni bir repository oluşturuyoruz. "Repository" bizim dosyalarımızı tutacak, bizim için ayrılacak alana verilen isimdir.

    Tıkladıktan sonra karşımıza gelen sayfada "Repository Name" alanına projemizin adını boşluk bırakmadan yazıyoruz. "Description" kısmına ise projemizin açıklamasını yazabiliriz ya da boş bırakabiliriz. Projemiz Açık Kaynak Kodlu ise "Public" seçeneğini, kapalı ise "Private" seçiyoruz ve "Create Repository" butonuna tıklıyoruz.
